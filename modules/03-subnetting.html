Subnetting is the language of network boundaries. Without it, you cannot define firewall rules, map targets, or pivot through an enterprise network.

1. Binary Logic

An IPv4 address is just a 32-bit number hidden behind four octets. To understand masks, you must master the "Power of 2".

Conversion Tip

To convert decimal to binary, simply subtract the largest possible power of 2.
Example: 192 = 128 + 64. So in binary, it is 11000000.

2. Special IP Ranges

RFC 1918 defines the addresses we use internally. These are never "routable" on the public internet.

3. Subnetting & CIDR

CIDR (Classless Inter-Domain Routing) notation tells us how many bits are used for the Network vs the Host.

üß™ The "Minus 2" Rule

In every subnet, the first IP is the Network ID and the last IP is the Broadcast. You cannot assign these to a computer.
Formula: (2^bits) - 2 = Usable Hosts.

4. Network Segmentation

By splitting a large network (e.g., a /16) into many small subnets (e.g., /24s), admins can enforce security policies between departments.

üõ°Ô∏è Lateral Movement Defense

If a hacker compromises a PC in the "Guest Wi-Fi" subnet, they shouldn't be able to "see" the "Server Room" subnet. Proper subnetting creates Internal Choke Points for firewalls to inspect traffic.

üìå Module Recap

  • Binary is the foundation of all subnet masking.
  • Private IPs stay internal; Public IPs are internet-facing.
  • CIDR notation (like /24) is a shorthand for the subnet mask.
  • Segmentation uses subnets to isolate sensitive data from common users.
