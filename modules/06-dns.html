DNS & Name Resolution

The Domain Name System (DNS) is the phonebook of the internet. It translates human-friendly names like google.com into machine-ready IP addresses.

1. How DNS Works

DNS is a distributed, hierarchical database. No single server holds all the data. Instead, authority is delegated down a chain.

üîç Recursive vs. Iterative

Recursive: Your computer asks the Resolver (e.g., 8.8.8.8) to do all the work and just come back with the answer.
Iterative: The Resolver asks each level of the hierarchy itself until it finds the target.

2. DNS Record Types

DNS doesn't just store IPs. It stores various records that define how a domain behaves.

3. The Resolution Process

Launch DNS Simulation

When you type a URL, a specific sequence of events occurs to find the IP:

  1. Local Cache: Check if you've visited recently.
  2. Hosts File: Check the local /etc/hosts (Linux) or C:\Windows\System32\drivers\etc\hosts (Win).
  3. Recursive Resolver: Ask your ISP or a public resolver (Cloudflare/Google).
  4. The Hierarchy: The resolver queries Root, TLD, and Authoritative servers.

Hackers often modify the local hosts file during an infection to redirect bankofamerica.com to their own malicious IP, bypassing DNS entirely.

4. Recon Tooling: nslookup & dig

Pentesters use these tools to map out a target's infrastructure before an attack.

Using nslookup

C:\> nslookup google.com
Server: UnKnown
Address: 192.168.1.1
Non-authoritative answer:
Name: google.com
Address: 142.250.190.46

Using dig (The Pro Choice)

root@kali:~# dig google.com MX +short
10 smtp.google.com.
// This finds the mail servers for the target instantly.

üß™ Mini Exercise

Try to find the TXT records for github.com using dig github.com TXT. You will likely see "verification" strings for various services they use.

5. DNS Attack Vectors

DNS is often unprotected and high-trust, making it a favorite for attackers.

üíÄ DNS Poisoning (Spoofing)

Flooding a resolver with fake entries so that users are sent to a malicious site. This is a classic Man-in-the-Middle technique.

üì° DNS Tunneling

Because DNS traffic is rarely blocked by firewalls, hackers hide data inside DNS queries (TXT records) to exfiltrate passwords or receive commands from a C2 server.

üìå Module Recap

  • DNS translates names to IPs in a hierarchical fashion.
  • A records are for IPv4; MX for mail; TXT for verification.
  • dig and nslookup are essential for infrastructure discovery.
  • DNS Poisoning and Tunneling are critical security risks.
